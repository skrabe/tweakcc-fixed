## 2. The pivotal discovery (why the integration target is `grep`/ugrep, not `rg`)

The journey corrected several wrong assumptions. Post-compaction, DO NOT relitigate these — they're verified:

1. **CC 2.1.186 ships NO Grep/Glob tool to the model.** Verified authoritatively via a MITM proxy capturing the real API `tools[]` (no Grep/Glob; 10 tools user-env / 32 plain-env) AND the harness error: _"No such tool available: Grep. Grep is not available in this session — search file contents with `grep` via the Bash tool instead."_ The gate is `Ait()`/`rv()`/`zc()` (the "search-tools" mechanism), **hardcoded-on via `ot("true")`**, on macOS (`zc()`), when `searchToolsOptIn` is false (default) and entrypoint≠`local-agent`. It is **NOT** `ENABLE_TOOL_SEARCH` (that only defers MCP/`shouldDefer` tools; Grep/Glob have no `shouldDefer` and are excluded entirely → ToolSearch can't find them either). NOT introduced by tweakcc (identical pristine vs patched). Grep/Glob return only for subagents (`entrypoint==="local-agent"`) or when a prompt names them (`searchToolsOptIn→true`).
   - ⇒ The Grep-tool-backed design (resolver repoint + Grep-description guidance) is **bypassed for the main agent**. (It still fires for subagents — see §6 "rg/Grep secondary".)

2. **CC SHADOWS `grep`→embedded `ugrep` and `find`→embedded `bfs` in its shell snapshot.** When the model runs `grep …`, CC's snapshot-injected wrapper function runs:

   ```sh
   grep () {
     local _cc_a; for _cc_a in "$@"; do case "$_cc_a" in
       (-*-filter*|-*-pager*|-*-view*|-*-format-open*|-*-config*|---*|-@*|-*-save-config*|-[Zz]*|-[!-]*[Zz]*|--null|--null-data) command grep "$@"; return ;;  # unsupported → real grep
     esac; done
     local _cc_bin="${CLAUDE_CODE_EXECPATH:-}"; [[ -x $_cc_bin ]] || _cc_bin=/Users/batricperovic/.local/bin/claude
     if [[ ! -x $_cc_bin ]]; then command grep "$@"; return; fi
     ARGV0=ugrep "$_cc_bin" -G --ignore-files --hidden -I --exclude-dir=.git --exclude-dir=.svn --exclude-dir=.hg --exclude-dir=.bzr --exclude-dir=.jj --exclude-dir=.sl "$@"   # (zsh/win32; else: exec -a ugrep "$_cc_bin" …)
   }
   ```

   i.e. the **claude binary dispatched as `ugrep`** (Bun multicall, like `argv0=rg`→embedded ripgrep). `find` is the same with `bfs`. This wrapper lives in `~/.claude/shell-snapshots/snapshot-zsh-*.sh` — **CC's own snapshot, NOT the user's `.zshrc`.** So it's CC-scoped: the user's terminal `grep`/`find` are the real system ones, untouched. (ugrep = a grep-compatible "Universal grep" bundled in the claude binary.)

3. **EMPIRICAL usage (the user's 12,680 transcripts, 214,423 Bash commands):** `grep` = **40,116**, `rg` = **294**, `find` = 8,581. The model uses **grep ~136× more than rg.** So **`grep`/ugrep is THE target.** rg is a rounding error (but we patch it too — free win, §6).

4. **The grep wrapper is generated by cli.js** (the snapshot-generator), as a string array. The repoint anchor (verified):
   ```
   ...["function ${e} {", ...argCheckLoop,
       `  local _cc_bin="\${${UZr}:-}"`,                 // UZr = "CLAUDE_CODE_EXECPATH"
       `  [[ -x $_cc_bin ]] || _cc_bin=${Oa([a])}`,       // a = claude bin path
       `  if [[ ! -x $_cc_bin ]]; then command ${e} \${1+"$@"}; return; fi`,
       "  if [[ -n ${ZSH_VERSION:-} ]]; then",
       `    ARGV0=${t} "$_cc_bin" ${o}`,                  // t = "ugrep"|"bfs", o = flags  ← REPOINT HERE
       '  elif [[ "$OSTYPE" == "msys" ]] || ... ; then',
       `    ARGV0=${t} "$_cc_bin" ${o}`,
       "  else",
       `    (exec -a ${t} "$_cc_bin" ${o})`,              // ← AND HERE
   ...]
   ```
   `e`=command (`grep`/`find`), `t`=embedded tool (`ugrep`/`bfs`), `o`=flags (`-G --ignore-files --hidden -I --exclude-dir=.git …` + `"$@"`). Comment in snapshot: `# Shadow find/grep with embedded bfs/ugrep`.

## 3. THE ARCHITECTURE (final)

**Repoint the grep→ugrep + find→bfs shadow to our `fff` wrapper.** Transparent (the model's 40k `grep` calls become fff with zero compliance — the registry-level robustness opencode #6506 demands), CC-scoped by construction (the snapshot is CC's → terminal untouched, no `CLAUDECODE` check needed), high-coverage (hits the tool actually used).

### 3.1 The patch (primary)

In cli.js's snapshot-generator, swap `"$_cc_bin"` → our fff wrapper path in the `ARGV0=${t} "$_cc_bin" ${o}` lines (zsh + win32 branches) and `exec -a ${t} "$_cc_bin" ${o}` (else). Keep `ARGV0=${t}` so the wrapper knows the mode (ugrep→content, bfs→filename). Anchor on `ARGV0=${t} "$_cc_bin" ${o}` and `exec -a ${t} "$_cc_bin" ${o}` (string literals in cli.js; minified-id-free; validate on linux bundles). The wrapper path is injected at apply-time. Idempotency + null-on-fail per the patch-creation skill. Gated on the toggle.

- The claude bin (`CLAUDE_CODE_EXECPATH` / `~/.local/bin/claude`) is still in the wrapper's env → the wrapper uses it for FALLBACK (`exec -a ugrep "$claudebin" …` to the real embedded ugrep/bfs).

### 3.2 The `fff` wrapper (multicall, grep/ugrep + bfs + rg compatible)

Rust binary, statically links `fff-search = "0.9.6"` (crates.io; crate name `fff-search`, lib `fff`). Dispatches on `argv0`:

- **`ugrep`** (the grep path, 99% of traffic): parse grep/ugrep flags (`-G` basic-regex, `-r -n -l -i -c -w -E -o --include --exclude --exclude-dir --ignore-files --hidden -I` + the model's args). Route **literal/identifier content searches → fff `content_search` (PlainText)**, ranked/def-first; **regex / unsupported → fall back to real embedded ugrep** (`exec -a ugrep "$claudebin" <orig args>`).
- **`bfs`** (the find path): filename search → fff `find_files` (fuzzy/frecency) for eligible; else fall back to real bfs.
- **`rg`** (secondary, §6): rg-compatible (the earlier-built behavior) → fff or embedded rg.
- **`fff`** (explicit, for `--fuzzy`/discovery if we also expose it): always fff.
- **Output:** agent-tuned text by default — reuse **fff-mcp's `GrepFormatter`** (`crates/fff-mcp/src/output.rs`, the agent-proven formatter): read-hint line (`→ Read path (reason)`), `N/M matches shown`, frecency-ranked def-first groups, `[def]` tags, definition auto-expansion, cursor pagination. `--json` → rg/grep-parseable for programs. (The model reads this directly — richer is better.)
- **Fallback:** never silent; log to stderr; `--no-fallback` for CI.
- **Exit codes:** 0 match / 1 no-match / 2 error (grep/rg semantics).
- **`--fuzzy`:** neo_frizbee Smith-Waterman; default exact; auto zero-result recovery ladder (broaden→fuzzy→path-suggest, mirroring fff-mcp `perform_grep`).

### 3.3 VERIFIED correctness facts (do not re-doubt)

- **fff `content_search` PlainText == ripgrep set-equivalent**: harness proved 205/205 and 4847/4847 keys (incl. `const`/`export`/`function`) with **0 divergence**. The "fff indexes symbols not keywords, falls back to rg for keywords" claim is an **MCP-tool/index-ranking artifact, NOT the library `content_search`** — our wrapper uses the library, so NO keyword fallback needed. (Re-confirm in the shadow harness; but my own test contradicts the general claim.)
- **embedded re-exec works**: `spawn(claudebin, [...], {argv0:"rg"})` → `ripgrep 14.1.1`; correct matches. Same mechanism works for `ugrep`/`bfs` (the shadow wrapper already uses it). So fallback is solid.
- **`fff-search` builds locally** (needs cmake for libgit2-sys; installed). Cross-build via **cargo-zigbuild** (fff's own release.yaml template) for `x86_64-unknown-linux-musl`, `aarch64-unknown-linux-musl`, `aarch64-apple-darwin`.

## 10b. Snapshot mechanics (VERIFIED this session — load-bearing)

- The grep/find shadow lives in a **per-session** shell snapshot `~/.claude/shell-snapshots/snapshot-${shell}-${Date.now()}-${rand}.sh`, generated by cli.js's `nJd(e,l,o)` then sourced via `execFile(shell,["-c","-l",content])`.
- **Generated FRESH per session, LAZILY on first Bash-tool use** (NOT at startup; NOT reused). `P2m()` is a janitor that prunes old `.sh` files, not a reuse path. ⇒ **No snapshot-clearing needed on apply**: the next interactive session regenerates from the patched generator automatically. (Earlier "must clear" hypothesis was wrong.)
- **`--print` (headless) does NOT generate/use the snapshot** — it ran a Bash command but created no snapshot. So `--print` grep ≠ the shadow; the feature targets **interactive TUI sessions** (where the 40k greps happen). Verify live with a real PTY turn, never `--print` (see memory `reference_print_smoke_misses_interactive_prompt_bugs`).
- **Install-refresh trap (FIXED):** `ensureRgFffWrapper` short-circuited on `dest` existence and never redeployed a rebuilt wrapper → the patch pointed at a stale binary (emitted `path:content`, no `src/`/line-numbers). Fix: repo-local build is canonical + refreshed by mtime; npx path re-fetches on version change (`.version` stamp); atomic install (temp+rename, fresh inode). After ANY wrapper rebuild, re-`--apply` redeploys it. Verify: `cmp installed release`.
